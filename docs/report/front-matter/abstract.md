# ABSTRACT

This study presents the design of a Secure Cloud-Based Patient Data Management System for Alamin Clinic, a private healthcare provider in Saudi Arabia whose on-premise infrastructure was compromised by a ransomware attack that resulted in the complete encryption of patient records and the disruption of clinical operations. The study identifies four systemic deficiencies in the existing infrastructure — flat network architecture, manual deployment processes, absence of data encryption, and no operational recovery plan — that collectively enabled the attack and that are representative of the security posture of small private healthcare providers operating on legacy on-premise systems. To address these deficiencies, the proposed system adopts a three-tier architecture deployed on Amazon Web Services, separating the presentation, application, and database layers across isolated Virtual Private Cloud subnets, with the database tier maintaining no direct internet access path. Infrastructure as Code using Terraform enables automated and reproducible deployment and rapid recovery from infrastructure failure, while a DevSecOps continuous integration and delivery pipeline incorporating Trivy, SonarQube, and Checkov enforces automated vulnerability scanning at every code commit, blocking deployment on any critical security finding. Identity and Access Management policies and Role-Based Access Control enforce least-privilege access across three user roles — Doctor, Admin, and Patient — at both the application and database levels, with PostgreSQL row-level security providing independent data isolation enforcement at the storage layer. HIPAA compliance posture is measured continuously through AWS Security Hub. The complete system design, encompassing network topology, security group configuration, database schema with row-level security policies, DevSecOps pipeline specification, and interface wireframes, establishes the full architectural blueprint from which the PSM2 implementation phase will proceed, with defined evaluation targets of a Recovery Time Objective under fifteen minutes and a passing HIPAA security posture score as measured by AWS Security Hub.

# ABSTRAK

Kajian ini membentangkan reka bentuk Sistem Pengurusan Data Pesakit Berasaskan Awan yang Selamat untuk Klinik Alamin, sebuah pembekal penjagaan kesihatan swasta di Arab Saudi yang infrastruktur premisnya telah diserang oleh perisian tebusan, menyebabkan penyulitan keseluruhan rekod pesakit dan gangguan kepada operasi klinikal. Kajian ini mengenal pasti empat kelemahan sistemik dalam infrastruktur sedia ada — seni bina rangkaian rata, proses penggunaan secara manual, ketiadaan penyulitan data, dan tiada pelan pemulihan operasi — yang secara kolektif membolehkan serangan tersebut berlaku dan yang mewakili kelemahan postur keselamatan pembekal penjagaan kesihatan swasta kecil yang masih beroperasi menggunakan sistem premis warisan. Bagi menangani kelemahan ini, sistem yang dicadangkan menggunakan seni bina tiga peringkat yang digunakan pada Amazon Web Services, memisahkan lapisan pembentangan, aplikasi, dan pangkalan data merentasi subnet Awan Peribadi Maya yang terpencil, dengan peringkat pangkalan data mengekalkan tiada laluan capaian internet secara langsung. Infrastruktur sebagai Kod menggunakan Terraform membolehkan penggunaan secara automatik dan boleh dihasilkan semula serta pemulihan pantas daripada kegagalan infrastruktur, manakala saluran paip penghantaran dan integrasi berterusan DevSecOps yang menggabungkan Trivy, SonarQube, dan Checkov menguatkuasakan pengimbasan kerentanan secara automatik pada setiap komit kod, dengan menyekat penggunaan apabila terdapat sebarang penemuan keselamatan kritikal. Dasar Pengurusan Identiti dan Akses serta Kawalan Akses Berasaskan Peranan menguatkuasakan akses dengan keistimewaan minimum merentasi tiga peranan pengguna — Doktor, Pentadbir, dan Pesakit — pada peringkat aplikasi dan pangkalan data, dengan keselamatan peringkat baris PostgreSQL menyediakan penguatkuasaan pengasingan data secara bebas pada lapisan storan. Pematuhan HIPAA dinilai secara berterusan melalui AWS Security Hub. Reka bentuk sistem yang lengkap, merangkumi topologi rangkaian, konfigurasi kumpulan keselamatan, skema pangkalan data dengan dasar keselamatan peringkat baris, spesifikasi saluran paip DevSecOps, dan lakaran antara muka, menetapkan pelan seni bina penuh dari mana fasa pelaksanaan PSM2 akan diteruskan, dengan sasaran penilaian yang ditakrifkan iaitu Objektif Masa Pemulihan di bawah lima belas minit dan skor postur keselamatan HIPAA yang lulus sebagaimana diukur oleh AWS Security Hub.
